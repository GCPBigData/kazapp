Complexidade O(n)

A complexidade O(n) é uma das mais fáceis de entender, e será a quarta notação que estudaremos.

Como sabemos, "n" significa quantidade de dados de entrada, assim dizemos que a complexidade O(n) é aquela que apresentará em seu pior caso, o valor de iterações referente ao tamanho dos dados de entrada.

Um bom exemplo para essa complexidade seria um algoritmo de busca sequencial, onde iteramos sob uma lista de dados, elemento a elemento, e verificamos se o valor atual é o que estamos procurando. O pior caso seria quando o elemento procurado fosse o último valor da lista, que seria correspondente ao valor "n" da quantidade de dados de entrada.

A performance desse algoritmo é boa, pois seu crescimento se baseará na quantidade de dados de entrada para o pior caso, sendo um algoritmo interessante mesmo para o processamento de grandes quantidades de dados.

Complexidade O(n)

Vamos ver como seria uma implementação de um exemplo na prática.

No algoritmo a seguir, implementamos uma busca sequencial, que basicamente itera sob um lista de dados, e verifica item a item se ele é igual ao que estamos buscando.


private int executar(int[] numeros, int numero) {
    for (int i=0; i < numeros.length; i++) {
        if (numeros[i] == numero) {
            return i;
        }
    }
    return -1;
}

Repare que o método acima retorna o índice com a posição do elemento na lista de entrada, ou "-1" caso o elemento não esteja presente.

Vamos praticar?

Para você testar na prática e verificar a performance de um algoritmo O(n), vamos criar uma classe contendo um experimento que demonstrará a performance do algoritmo de busca sequencial.

Para isso crie um arquivo chamado OrdemN.java com o seguinte código:


public class OrdemN {

    private int executar(int[] numeros, int numero) {
        for (int i=0; i < numeros.length; i++) {
            if (numeros[i] == numero) {
                return i;
            }
        }
        return -1;
    }

    private static void executarExperimento(OrdemN ordemN, int qtdNumeros, int numero) {
        int[] numeros = new int[qtdNumeros];
        for (int i=0; i < qtdNumeros; i++) {
            numeros[i] = i + 1;
        }
        long inicio = System.currentTimeMillis();
        System.out.println("Executando experimento para lista de " + qtdNumeros + " números.");
        ordemN.executar(numeros, numero);
        long fim = System.currentTimeMillis();
        System.out.println("Experimento executado em " + (fim - inicio) + "ms\n");
    }

    public static void main(String[] args) {
        OrdemN ordemN = new OrdemN();
        executarExperimento(ordemN, 1, 1);
        executarExperimento(ordemN, 100, 100);
        executarExperimento(ordemN, 1000, 1000);
        executarExperimento(ordemN, 100000, 100000);
        executarExperimento(ordemN, 1000000, 1000000);
        executarExperimento(ordemN, 10000000, 10000000);
        executarExperimento(ordemN, 100000000, 100000000);
        executarExperimento(ordemN, 200000000, 200000000);
        executarExperimento(ordemN, 300000000, 300000000);
        executarExperimento(ordemN, 350000000, 350000000);
    }

}

O código acima chama o mesmo método estudado anteriormente, porém com diferentes valores de entrada, e sempre buscando pelo último elemento da lista, pois assim medimos sua performance no pior caso, além de registrar o tempo de execução para cada chamada.
