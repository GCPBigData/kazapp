Complexidade quadrática O(n2)

A complexidade quadrática O(n2) é a segunda notação que estudaremos.

Na matemática dizemos que um número quando elevado a potência 2, será multiplicado por ele mesmo uma vez, resultando em valor que tem um crescimento substancial a medida que o valor de entrada aumenta.

Ele é representado pelo potência de "2", por isso dizemos O(n2), que significa notação Big-O com complexidade quadrática.

Embora os algoritmos dessa categoria sejam bem melhores dos que os de complexidade fatorial estudados anteriormente, ele ainda assim deixa muito a desejar ao manipularmos grandes quantidades de dados.

Vamos olhar e comparar alguns casos para entender seu comportamento:

  • 02 -> 0 * 0 = 0
  • 12 -> 1 * 1 = 1
  • 22 -> 2 * 2 = 4
  • 32 -> 3 * 3 = 9
  • 42 -> 4 * 4 = 16
  • 52 -> 5 * 5 = 25
  • 62 -> 6 * 6 = 36
  • 72 -> 7 * 7 = 49
  • 82 -> 8 * 8 = 64
  • 92 -> 9 * 9 = 81
  • 102 -> 10 * 10 = 100

Conforme você reparou, os números crescem de modo significativo entre eles.

Repare que para 1 seu valor é também 1, enquanto que de 10 é 100, ou seja, 100 vezes maior para uma diferença de 10 números apenas.

O que eu quero mostrar com isso é justamente a velocidade com que os resultados crescem a medida que adicionamos um único número.

Uma carecterística para identificar um algoritmo quadrático, é reparar se ele possui dois laços for encadeados, pois assim sua execução crescerá exponencialmente, conforme veremos ainda nesta lição.

Portanto, um algoritmo de complexidade quadrática é um algortimo que em seu pocessamento cresce de modo significativo entre pequenos valores, o que significa que ele se tornará inviável ao tentarmos executá-lo com valores muito grandes.

Concluímos que esse tipo de algoritmo é limitado a resolver problemas com poucos dados de entrada, pois ele terá uma performance muito ruim para grandes quantidades de dados.

Complexidade quadrática O(n2)

Agora que você já sabe o que é um algoritmo de complexidade quadrática, vamos ver como seria uma implementação do mesmo na prática:


private void algoritmoComplexidadeQuadratica(int numero) {
    for (int i=0; i < numero; i++) {
        for (int j=0; j < numero; j++) {
            System.out.println(i + ", " + j);
        }
    }
}

Repare que o método acima é bastante simples, porém tem uma performance ruim a medida que incrementamos nosso valor de entrada.

Isso ocorre porque o método encadeia dois laços for, o que faz com que seu processamento cresça de modo exponencial, o que deverá ser evitado para soluções que contenham grandes quantidades de dados a serem processadas.

Por outro lado, é completamente aceitável utilizar soluções como essa quando apenas precisamos processar pequenos valores, pois são simples de implementar e terão uma performance aceitável.

Vamos praticar?

Para você testar na prática e verificar a performance de um algoritmo quadrático, vamos criar uma classe contendo um experimento que demonstrará a performance do algoritmo estudado anteriormente.

Para isso crie um arquivo chamado OrdemQuadratica.java com o seguinte código:


public class OrdemQuadratica {

    private static long numIteracoes;

    private void executar(int numero) {
        for (int i=0; i < numero; i++) {
            for (int j=0; j < numero; j++) {
                OrdemQuadratica.numIteracoes++;
            }
        }
    }

    private static void executarExperimento(OrdemQuadratica ordemQuadratica, int numero) {
        ordemQuadratica.numIteracoes = 0;
        long inicio = System.currentTimeMillis();
        System.out.println("Executando experimento para número = " + numero);
        ordemQuadratica.executar(numero);
        long fim = System.currentTimeMillis();
        System.out.println("Experimento iterado " + OrdemQuadratica.numIteracoes + " vezes.");
        System.out.println("Experimento executado em " + (fim - inicio) + "ms\n");
    }

    public static void main(String[] args) {
        OrdemQuadratica ordemQuadratica = new OrdemQuadratica();
        executarExperimento(ordemQuadratica, 1);
        executarExperimento(ordemQuadratica, 50);
        executarExperimento(ordemQuadratica, 100);
        executarExperimento(ordemQuadratica, 1000);
        executarExperimento(ordemQuadratica, 5000);
        executarExperimento(ordemQuadratica, 10000);
        executarExperimento(ordemQuadratica, 100000);
        executarExperimento(ordemQuadratica, 1000000);
    }

}

O código acima chama o mesmo método estudado anteriormente, porém com diferentes valores de entrada, e registra o tempo de execução para cada chamada, assim como a quantidade de iterações por execução.
